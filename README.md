# 招標網資料自動抓取器 (Procurement Data Scraper)

這是一個能讓您輸入「政府電子採購網」等標案網址後，自動將標案的相關資訊（如機關名稱、得標廠商、決標金額等）抓取並直接寫入您個人 Google 試算表的小型網頁應用程式。

## 系統架構

1. **前端網頁 (Frontend)**：
   - 採用 **Glassmorphism (玻璃擬物化)** 設計風格的現代化介面，提供輸入網址與查看歷史紀錄的功能。
   - **歷史紀錄總表**：包含即時連動、快取秒開、多種日期排序功能。
   - **互動式狀態管理**：可直接在網頁核取「結束」狀態，自動雙重確認並同步至雲端試算表。
   - **響應式全螢幕設計**：自適應螢幕寬度（高達 1600px），並支援手機橫向滑動查看表格。
   - 由 `index.html`, `style.css`, `script.js` 組成，可部署於任何靜態網頁伺服器（推薦 GitHub Pages 或是直接開啟本地網頁）。

2. **後端代理服務 (Backend / GAS proxy)**：
   - 利用 **Google Apps Script (GAS)** 避開了瀏覽器惱人的跨網域限制 (CORS 問題)。
   - **智慧欄位對應**：會自動去讀取您的試算表「第一列(表頭)」目前有哪些名稱，並智慧填寫進去，因此更改您的試算表欄位順序也**完全不會壞掉**。支援相容多種常見欄位寫法（如「招標網址」、「網址」等）。
   - 由 `GAS_Script.js` 裡面的程式碼驅動，當作 API 給前端呼叫。

## 部署教學（當 GAS 程式碼有更新時）

只要您的 `GAS_Script.js` 程式碼有任何變更或升級，您都**必須**回到您的 Google 試算表中進行更新：
1. 點擊試算表上方的「擴充功能」 > 「Apps Script」。
2. 將 `GAS_Script.js` 的所有文字複製，貼上並覆蓋掉裡面的程式碼。
3. 點擊右上角「部署」 > 「管理部署作業」。
4. 點擊彈出視窗右上方的「編輯 (鉛筆圖示)」。
5. **在版本的地方點開下拉選單，選擇「建立新版本」。**
6. 點擊「部署」。

> *沒有建立新版本的話，Google 伺服器會一直執行您的舊時代程式碼！這點非常重要！*
