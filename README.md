# 招標網資料自動抓取器 (Procurement Data Scraper)

這是一個能讓您輸入「政府電子採購網」等標案網址後，自動將標案的相關資訊（如機關名稱、得標廠商、決標金額等）抓取並直接寫入您個人 Google 試算表的小型網頁應用程式。

## 系統架構

1. **前端網頁 (Frontend)**：
   - 採用 **Glassmorphism (玻璃擬物化)** 設計風格的現代化介面，提供輸入網址與查看歷史紀錄的功能。
   - **歷史紀錄總表**：包含即時連動、快取秒開、多種日期排序功能。
   - **互動式狀態管理**：可直接在網頁核取「結束」狀態，自動雙重確認並同步至雲端試算表。
   - **響應式全螢幕設計**：自適應螢幕寬度（高達 1600px），並支援手機橫向滑動查看表格。
   - 由 `index.html`, `style.css`, `script.js` 組成，可部署於任何靜態網頁伺服器（推薦 GitHub Pages 或是直接開啟本地網頁）。

2. **後端代理服務 (Backend / GAS proxy)**：
   - 利用 **Google Apps Script (GAS)** 避開了瀏覽器惱人的跨網域限制 (CORS 問題)。
   - **智慧欄位對應**：會自動去讀取您的試算表「第一列(表頭)」目前有哪些名稱，並智慧填寫進去，因此更改您的試算表欄位順序也**完全不會壞掉**。支援相容多種常見欄位寫法（如「招標網址」、「網址」等）。
   - 由 `GAS_Script.js` 裡面的程式碼驅動，當作 API 給前端呼叫。

## 建立與更新日誌 (Changelog)

這份專案是在經過多次優化後成型的。以下是主要的演進紀錄：

### 核心功能建立
- [x] **建立基礎架構**：完成前後端分離設計，使用 GAS 作為代理伺服器抓取原始 HTML，以此繞過前端網頁的 CORS 限制。
- [x] **動態表頭匹配機制**：GAS 腳本自動讀取 Google Sheet 第一列的文字，不綁死固定欄位，試算表可以隨意移動欄位順序。
- [x] **爬蟲邏輯強化**：針對特定的數字欄位（如「決標金額」）加入 Regular Expression 處理，剔除中文字只保留「15,980,000」這樣的純數字與逗號格式。

### 前端體驗升級 (UI/UX)
- [x] **歷史資料連線 (GET 請求)**：新增讀取 API，讓試算表上的資料可以直接匯出顯示於前端首頁。
- [x] **快取秒開技術 (LocalStorage)**：打開網頁或重新整理時，優先載入上一次瀏覽器記錄的舊資料快取，背景再默默更新最新資料，達成「零等待」的使用體驗。
- [x] **全螢幕延展彈性佈局**：移除不必要的中間預覽表格，並將畫面可用寬度拉滿（依賴解析度彈性最高達 1600px寬），改善資料擠成一團的問題。
- [x] **前端自訂排序功能**：新增右側下拉選單，自動擷取「115/01/22」這樣的民國年份字串，轉換為程式可讀時間序列，支援公告日、履約日期的最新或最舊排序。
- [x] **響應式手機版設計 (RWD)**：增加 `@media` 查詢，螢幕過小時表格將轉為可橫滑捲動的模式，版面不再被生硬壓縮。

### 高階與進階防呆機制
- [x] **自動公式注入 (XLOOKUP)**：抓取新標案存入試算表時，GAS 將自動計算當前列數，並在「業務」欄位注入 `=XLOOKUP()` 公式。
- [x] **「結束」狀態互動式打勾**：
    - 前端在抓取到資料且該資料的「結束」欄位為未核取 (`FALSE`) 時，會直接在畫面上繪製出真正的 HTML Checkbox。
    - 使用者打勾時會跳出 double-click 確認視窗防呆。
    - 確認後呼叫新增的一支 **Action API (`update_status`)**，遠端更新試算表把那一格打勾。
    - 前端網頁再次重整後會自動濾除（隱藏）所有被標記為 `TRUE` （已結束）的資料。

### 選單排序與除錯優化 (2026-03-03 近期更新)
- [x] **精簡設定單純化與預設優化**：依照您的指定，將長長一串排序列表精簡為明確的「5種標準配置」(依照 存入時間/公告日/履約起迄日期 最新和最舊、以及機關和廠商地址的前3碼順序)，確保預設資料會最新顯示在前方。
- [x] **修復預設圖表空白問題**：透過嚴格修正腳本迴圈括號(`script.js` line 226)的包裹作用域，讓選取「存入時間(預設)」時，迴圈能脫險正常載入全部列表資訊！

## 部署教學（當 GAS 程式碼有更新時）

只要您的 `GAS_Script.js` 程式碼有任何變更或升級，您都**必須**回到您的 Google 試算表中進行更新：
1. 點擊試算表上方的「擴充功能」 > 「Apps Script」。
2. 將 `GAS_Script.js` 的所有文字複製，貼上並覆蓋掉裡面的程式碼。
3. 點擊右上角「部署」 > 「管理部署作業」。
4. 點擊彈出視窗右上方的「編輯 (鉛筆圖示)」。
5. **在版本的地方點開下拉選單，選擇「建立新版本」。**
6. 點擊「部署」。

> *沒有建立新版本的話，Google 伺服器會一直執行您的舊時代程式碼！這點非常重要！*
